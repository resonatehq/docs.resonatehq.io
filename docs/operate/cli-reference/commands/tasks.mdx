Interact with the Task Framework to simulate Application Node behavior.

```shell title="Task commands"
resonate tasks <subcommand> [args] [flags]
```

## Available subcommands

- `claim` - Claim one or more tasks by ID
- `complete` - Mark a task as complete
- `heartbeat` - Send heartbeat signals for tasks

## Quick examples

```shell title="Claim a task"
resonate tasks claim task-123 --counter 1
```

```shell title="Complete a task"
resonate tasks complete task-123 --counter 1 --process-id worker-1
```

```shell title="Send heartbeat"
resonate tasks heartbeat task-123 --process-id worker-1
```

## Learn more

Run `resonate tasks --help` to see all subcommands.

Run `resonate tasks <subcommand> --help` for detailed usage.

:::tip For advanced users
The tasks command is primarily for testing and debugging the Task Framework. Most users interact with tasks through the SDK, not the CLI.
:::
