Start the Resonate (Durable Promise) Server.

```shell title="Start server"
resonate serve [flags]
```

:::tip Development shortcut
Use [`resonate dev`](./dev) for local development with in-memory storage. For production, use `resonate serve` with persistent storage.
:::

## Quick examples

```shell title="Start with persistent SQLite"
resonate serve --aio-store-sqlite-path /var/lib/resonate/resonate.db
```

```shell title="Start with PostgreSQL"
resonate serve \
  --aio-store-postgres-enable \
  --aio-store-postgres-host localhost \
  --aio-store-postgres-database resonate \
  --aio-store-postgres-username resonate \
  --aio-store-postgres-password secret
```

```shell title="Use a configuration file"
resonate serve --config resonate.yaml
```

```shell title="Enable HTTP authentication"
resonate serve --api-http-auth admin=secret
```

## Configuration

The `serve` command accepts many configuration flags for storage, networking, authentication, and more.

Run `resonate serve --help` to see all available options.

See the [server configuration guide](/operate/run-server#customize-configuration) for detailed setup instructions.

See the [CLI reference flags](/operate/cli-reference/flags) for flag documentation.
