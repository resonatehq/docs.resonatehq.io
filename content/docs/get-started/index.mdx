**Durable execution means your functions survive crashes, restarts, and scaling — automatically.** Resonate handles retries, recovery, and state persistence so you can write normal async code.

Want to get started with Resonate?

We want you to have the best possible experience while getting started and using Resonate.
Therefore, what you do next really depends on where you are coming from and what you are trying to do.

- **[Quickstart](/get-started/quickstart)**: If you want to dive right in, then check out our [quickstart guide](/get-started/quickstart) to get a Resonate application up and running in minutes.

- **[Examples](/get-started/examples)**: If you are trying to solve a specific problem, start with an [example](/get-started/examples) that matches your use case — each one ships with a runnable repo and a walkthrough of the Resonate primitives in play.

And if you have experience with either Temporal or DBOS you may want to review our "coming from x" guides, which can be helpful in mapping concepts from those platforms to Resonate.

- [Coming from Temporal](/evaluate/coming-from/temporal)
- [Coming from Restate](/evaluate/coming-from/restate)
- [Coming from DBOS](/evaluate/coming-from/dbos)
- [Coming from BYODE (Building Your Own Durable Execution)](/evaluate/coming-from/byode)
